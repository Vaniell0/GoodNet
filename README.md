P2P-платформа на C++23. Шифрование, транспорт и маршрутизация из коробки. Пиши свой протокол — остальное уже работает.

## Проблема

Для веба есть платформы: gRPC, REST-фреймворки, очереди сообщений. Для P2P — ничего. Каждый раз разработчик с нуля склеивает криптографию, транспорт, обход NAT, framing, сериализацию. Инструменты существуют по отдельности, но собрать их вместе — задача уровня отдельного проекта. Порог входа высокий, и новые разработчики уходят делать очередной веб-сервис вместо того, чтобы создавать что-то новое.

## Что даёт GoodNet

- **Готовая платформа** — криптография, транспорт, маршрутизация, relay работают из коробки. Разработчик пишет handler с бизнес-логикой, а не networking boilerplate.
- **Открытая архитектура** — каждый слой доступен через plugin API. Нужен свой транспорт, свой handshake flow, своя логика relay — подключай как плагин.
- **Не навязывает абстракции** — в отличие от фреймворков, которые прячут сетевой стек за непрозрачным API, GoodNet даёт контроль над каждым уровнем.
- **Экспериментальная площадка** — проверить идею нового протокола можно за день на реальной шифрованной сети, а не писать год инфраструктуру.

## Направление

GoodNet стремится стать NetOS — сетевой операционной системой, которая сама выбирает оптимальный транспорт, маршрутизирует трафик и договаривается о capabilities между узлами.
