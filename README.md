# GoodNet (Alpha)

> **Универсальная «нервная система» для распределенных систем. Программная Cisco в каждом узле.**

## 🌐 Суть проекта

**GoodNet** — это не просто сетевой фреймворк. Это фундамент для создания симметричных, децентрализованных и максимально гибких сетей. В то время как мир уходит в тяжелые облачные абстракции, GoodNet возвращает контроль инженеру, предоставляя модульный конструктор для построения чего угодно: от IoT-шлюза до защищенного меш-маршрутизатора.

## 🧩 Философия

### 1. Симметрия узлов

Забудьте о жестком делении на «клиент» и «сервер». В GoodNet каждый экземпляр — это **Node (Узел)**. Он одновременно и слушатель, и инициатор. Роль узла в топологии определяется не кодом, а набором подключенных плагинов.

### 2. Полное разделение транспорта и логики

* **Connector** отвечает за то, *как* байты попадают в узел (TCP, UDP, Bluetooth, LoRa, Radio).
* **Handler** отвечает за то, *что* с ними происходит (Chat, Logger, Database, Proxy).
Смена транспорта — это замена одного `.so` файла. Бизнес-логика остается нетронутой.

### 3. Прямой доступ без «магии»

GoodNet не прячет сокеты за десятью слоями интерфейсов. Накладные расходы по сравнению с чистым **Boost.Asio** минимальны. Если вам нужен доступ к параметрам сокета на этапе `accept()` — вы его получаете.

### 4. Программная Cisco

Идея GoodNet — стать универсальным «умным кирпичом». Загрузили модуль маршрутизации — получили роутер. Загрузили модуль шифрования и мост — получили защищенный прокси.

## 🎯 Почему именно GoodNet?

В экосистеме C++ существует огромный разрыв между:

1. **Низкоуровневыми библиотеками** (Boost.Asio), где всё нужно писать вручную.
2. **Тяжелыми фреймворками** (POCO, Envoy, ZeroMQ), которые навязывают свои правила, скрывают транспорт и потребляют много ресурсов.

**GoodNet занимает эту пустую нишу.** Он дает **одновременно**:

* **Raw-контроль:** Возможность отвергнуть соединение в первую миллисекунду.
* **Безопасность:** Обязательная SHA-256 верификация плагинов перед загрузкой.
* **Воспроизводимость:** Благодаря **Nix**, сборка проекта на любой машине дает побайтово идентичный результат.

## 🚀 Текущие возможности (Alpha)

| Возможность | Описание |
| --- | --- |
| **Динамические плагины** | Загрузка Handlers и Connectors как `.so` в рантайме без перезагрузки ядра. |
| **SHA-256 Верификация** | Защита от подмены модулей: манифест проверяется перед `dlopen`. |
| **Асинхронность** | Вся маршрутизация построена на Boost.Asio strand — никакой блокировки потоков. |
| **Изоляция модулей** | Использование `RTLD_LOCAL` предотвращает конфликты символов между плагинами. |
| **Nix Ecosystem** | Использование Flakes для управления зависимостями и идеальной воспроизводимости. |

## 🗺 Путь к Beta: Куда мы движемся?

Сейчас GoodNet — это мощный скелет. На этапе беты и далее проект превратится в:

1. **Универсальный адаптер:** Автоматическое определение типа соединения и адаптивная маршрутизация.
2. **Multi-language SDK:** Возможность писать плагины на любом языке (через WASM или C-FFI), сохраняя скорость ядра на C++.
3. **NAT Traversal & Mesh:** Встроенные механизмы для соединения узлов, находящихся за глубокими NAT, и построение самоорганизующихся сетей.

## 🛠 Разработка

Проект использует **Nix** для обеспечения чистоты окружения.

```bash
# Вход в окружение разработки
nix develop

# Сборка проекта
nix build

```

**GoodNet** — это манифест свободы в мире, где всё больше связей контролируется посредниками. Мы возвращаем связь тем, кому она принадлежит — узлам.
